# 要件定義書

**プロジェクト名:** bookshelf-app
**文書作成日:** 2026年3月11日
**バージョン:** 1.0

---

## 1. プロジェクト概要

### 1.1 目的

本アプリケーションは、個人の書籍管理を目的としたWebアプリである。ブクログ等の既存サービスに類似した読書記録機能を基本とし、複数拠点での書籍所在管理および学術論文執筆を想定したプロジェクト別参照メモ機能を追加で備える。

### 1.2 背景

- 複数の生活拠点を持つユーザーが、どの拠点にどの本があるかを把握しにくいという課題を解決する。
- 論文・レポート執筆時に、書籍から抜き出したメモをプロジェクト単位で整理・参照できる環境を提供する。

### 1.3 対象ユーザー

- 複数拠点で生活する個人
- 書籍を研究・執筆活動に活用する学生・研究者
- 読書記録を継続的につけたいユーザー

## 3. 機能要件

### 3.1 ユーザー認証

#### 3.1.1 アカウント登録

- メールアドレスとパスワードでアカウントを作成できる。
- 登録後はログイン済み状態となり、マイ書棚画面へ遷移する。
- パスワードは8文字以上とする。

#### 3.1.2 ログイン

- 登録済みのメールアドレスとパスワードでログインできる。
- 認証に失敗した場合はエラーメッセージを表示する。

#### 3.1.3 ログアウト

- ログアウト操作によりセッションを終了し、ログイン画面へ遷移する。

#### 3.1.4 アクセス制御

- 未認証ユーザーは書棚・ロケーション・プロジェクト系の全ページにアクセスできない。
- 未認証でアクセスした場合はログイン画面へリダイレクトする。

---

### 3.2 書籍管理

#### 3.2.1 書籍の追加 (Google Books API 検索)

- タイトル、著者名、またはISBNでGoogle Books APIを検索できる。
- 検索結果一覧を表示し、各候補から書籍を選択して自分の書棚に追加できる。
- 追加時に取得する情報は以下の通りとする。

| フィールド | 取得元 |
|------------|--------|
| タイトル | Google Books API |
| 著者名 | Google Books API |
| 出版社 | Google Books API |
| 出版年 | Google Books API |
| ISBN | Google Books API |
| 表紙画像URL | Google Books API |

#### 3.2.2 書籍の手動登録

- Google Books APIで見つからない書籍を手動で登録できる。
- 入力項目: タイトル (必須)、著者名、出版社、出版年、ISBN、表紙画像URL。

#### 3.2.3 読書ステータス管理

書籍ごとに以下の3種類のステータスを設定できる。

| ステータスID | 表示名 |
|-------------|--------|
| `want_to_read` | 読みたい |
| `reading` | 読んでいる |
| `read` | 読んだ |

- ステータスは登録時に設定し、後から変更できる。

#### 3.2.4 評価 (星評価)

- 1から5の整数で星評価を付けられる。
- 評価は任意項目とする (未評価を許容する)。

#### 3.2.5 レビュー (感想テキスト)

- 書籍ごとに任意でテキスト形式のレビューを記入できる。
- レビューは後から編集・削除できる。

#### 3.2.6 書籍の削除

- 書棚から書籍を削除できる。
- 削除時は確認ダイアログを表示する。
- 書籍に紐づく参照メモも合わせて削除される。

#### 3.2.7 書籍一覧の表示とフィルタリング

書棚画面では以下の条件で書籍を絞り込み表示できる。

- 読書ステータスによるフィルタリング
- 保管ロケーションによるフィルタリング
- タイトル・著者名によるテキスト検索
- 複数条件の組み合わせが可能

---

### 3.3 ロケーション管理

複数拠点での書籍所在を管理するための機能。

#### 3.3.1 ロケーションの作成

- ロケーション名 (必須) と説明 (任意) を入力して作成できる。
- ロケーションの例: 「東京の自宅」「大阪のオフィス」「実家」

#### 3.3.2 ロケーションの編集・削除

- 作成済みロケーションの名前・説明を編集できる。
- ロケーションを削除した場合、そのロケーションに割り当てられていた書籍のロケーション設定はクリア (NULL) される。

#### 3.3.3 書籍へのロケーション割り当て

- 書籍登録時および書籍編集時にロケーションを選択できる。
- 1冊の書籍に対して1つのロケーションを設定できる。
- ロケーションは任意設定とする (未設定を許容する)。

#### 3.3.4 ロケーション別の蔵書数表示

- ロケーション一覧画面では、各ロケーションに何冊の書籍が割り当てられているかを表示する。

#### 3.3.5 ロケーション別の書籍フィルタリング

- 書棚画面でロケーションを選択することで、そのロケーションに属する書籍のみを表示できる。

---

### 3.4 プロジェクト別参照メモ管理

論文・レポート執筆等を想定した、書籍から抜き出したメモをプロジェクト単位で管理する機能。

#### 3.4.1 プロジェクトの作成

- プロジェクト名 (必須) と説明 (任意) を入力して作成できる。
- プロジェクトの例: 「卒業論文」「書評記事」「勉強会資料」

#### 3.4.2 プロジェクトの編集・削除

- 作成済みプロジェクトの名前・説明を編集できる。
- プロジェクトを削除した場合、そのプロジェクトに紐づく参照メモも合わせて削除される。

#### 3.4.3 参照メモの追加

- 参照メモは「プロジェクト + 書籍」の組み合わせに紐づいて作成される。
- 入力項目は以下の通りとする。

| フィールド | 必須 | 説明 |
|------------|------|------|
| メモ本文 | 必須 | 引用・要約・気づきなどのテキスト |
| ページ参照 | 任意 | 参照元ページ番号 (例: 「p.42」「pp.120-125」) |

#### 3.4.4 参照メモの編集・削除

- 作成済みの参照メモを編集できる。
- 参照メモを削除できる。削除時は確認ダイアログを表示する。

#### 3.4.5 プロジェクト詳細画面

- プロジェクトに紐づく全参照メモを表示する。
- メモは「書籍ごと」にグループ化して表示する。
- 各グループには書籍タイトル・著者名を見出しとして表示する。

## 6. 非機能要件

### 6.1 パフォーマンス

- 主要ページの初期表示を軽量に保ち、低速回線でも実用的な速度で表示できることを目指す。
- 画像はSupabaseまたは外部CDN (Google Books画像URL) から直接配信し、サーバーでの変換処理は最小限にとどめる。
- 不必要なクライアントサイドJavaScriptを排除し、Next.js App RouterのServer Componentを積極的に活用する。

### 6.2 レスポンシブデザイン

- スマートフォン (幅360px以上)、タブレット、デスクトップの各画面サイズで操作可能なUIを提供する。
- Tailwind CSS v4のブレークポイントを利用してレイアウトを制御する。

### 6.3 セキュリティ・データ分離

- SupabaseのRow Level Security (RLS) を全テーブルに対して有効化する。
- 各テーブルのポリシーは「`user_id` が認証中のユーザーIDと一致する行のみ SELECT / INSERT / UPDATE / DELETE を許可する」とする。
- これによりユーザー間のデータ漏洩を防止し、マルチユーザー環境でも安全に運用できる。

### 6.4 ホスティング・運用コスト

- フロントエンドはVercelの無料プラン (Hobby) でホスティングする。
- データベース・認証・ストレージはSupabaseの無料プランで運用する。
- 月間の想定アクティブユーザー数が少数 (個人利用) である限り、無料枠内で継続運用できる構成とする。

### 6.5 デプロイの容易性

- エンジニア以外がデプロイ・初期セットアップを行えるよう、操作手順をドキュメントとして整備する。
- 環境変数は `.env.local` ファイルと Vercel の Environment Variables 画面の2箇所のみで管理し、複雑なインフラ設定を不要とする。
- Supabaseのマイグレーションは SQLファイルを Supabase Studio の SQL Editor に貼り付けるだけで完了できる形式で提供する。

## 7. 外部サービス連携

### 7.1 Google Books API

- エンドポイント: `https://www.googleapis.com/books/v1/volumes`
- 認証: APIキー方式 (環境変数 `GOOGLE_BOOKS_API_KEY` で管理)
- 利用用途: 書籍検索・書誌情報の自動取得
- APIキーはサーバーサイド (Next.js Route Handler) 経由でのみ使用し、クライアントに露出させない。

### 7.2 Supabase

- 利用サービス: Database (PostgreSQL)、Auth、Storage (将来拡張用)
- 接続情報は環境変数 `NEXT_PUBLIC_SUPABASE_URL` および `NEXT_PUBLIC_SUPABASE_ANON_KEY` で管理する。
- サーバーサイドの操作には `@supabase/ssr` パッケージのサーバークライアントを使用し、RLSが正しく機能する状態を維持する。
