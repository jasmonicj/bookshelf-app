# 引継ぎマニュアル — Bookshelf App

このマニュアルは、Bookshelf App を自分のサーバーで動かし、維持管理するための手順書です。
エンジニアでなくても、画面の説明通りに進めれば完了できます。
すべて無料のサービスを使います。

---

## このアプリについて

本の管理アプリです。読んだ本・読みたい本の記録、本棚の場所管理、読書メモのプロジェクト管理ができます。
ログインが必要で、自分のデータだけが表示されます。

---

## 必要なもの

- パソコン（作業はすべてブラウザで行います）
- メールアドレス（各サービスの登録に使います）
- 作業時間：初回のみ約1〜2時間

---

## 全体の流れ

1. GitHub アカウントを作り、アプリのコードをコピーする
2. Supabase でデータベースを作る
3. Vercel でアプリを公開する
4. Supabase の認証設定を完成させる
5. 動作確認をする

---

# パート 1: 初回セットアップ（一度だけ行う作業）

---

## ステップ 1: GitHub アカウントを作成し、リポジトリをフォークする

GitHub はプログラムのコードを保管する場所です。ここにアプリのコードが置かれています。

### 1-1. GitHub アカウントを作成する

1. ブラウザで `https://github.com` を開きます。
2. 右上の「Sign up」というボタンをクリックします。
3. 「Enter your email」と書かれた入力欄にメールアドレスを入力し、「Continue」をクリックします。
4. 次の画面でパスワードを入力し、「Continue」をクリックします。
5. ユーザー名（英数字で自分の名前など）を入力し、「Continue」をクリックします。
6. 「Would you like to receive product updates?」という質問が出ます。どちらを選んでも構いません。「Continue」をクリックします。
7. パズルの確認が表示されたら、指示に従って解きます。
8. 入力したメールアドレスに確認コード（数字8桁）が届きます。その数字を画面に入力します。
9. アカウントの種類を選ぶ画面では、無料の「Free」を選び、「Continue」をクリックします。
10. いくつかアンケートが表示されますが、すべて「Skip」でも問題ありません。

### 1-2. リポジトリをフォークする

「フォーク」とは、他の人のコードを自分のアカウントにコピーすることです。

1. ブラウザで `https://github.com/jasmonicj/bookshelf-app` を開きます。
2. 画面の右上付近に「Fork」というボタンがあります。クリックします。
3. 「Create a new fork」という画面が開きます。
4. 「Repository name」はそのまま「bookshelf-app」で構いません。
5. 下にある緑色の「Create fork」ボタンをクリックします。
6. 少し待つと、自分のアカウントにコピーされたページが開きます。
   - アドレスバーに `https://github.com/あなたのユーザー名/bookshelf-app` と表示されていれば成功です。

---

## ステップ 2: Supabase でデータベースを作成する

Supabase はデータベース（本のデータやユーザー情報を保存する場所）を提供するサービスです。

### 2-1. Supabase アカウントを作成する

1. ブラウザで `https://supabase.com` を開きます。
2. 右上の「Start your project」または「Sign In」ボタンをクリックします。
3. 「Continue with GitHub」というボタンをクリックします。
4. GitHub のログイン画面が開いた場合は、先ほど作ったアカウントのメールアドレスとパスワードでログインします。
5. 「Authorize supabase」という画面が表示されたら「Authorize supabase」ボタンをクリックします。
6. Supabase のダッシュボードが表示されれば登録完了です。

### 2-2. 新しいプロジェクトを作成する

1. Supabase のダッシュボードで「New project」ボタンをクリックします。
2. 「Organization」（組織名）を選択する欄があります。自動で作られているものをそのまま選択します。
3. 各項目を入力します。
   - 「Name」: プロジェクトの名前。例: `bookshelf-app`（英数字とハイフンのみ使えます）
   - 「Database Password」: データベースのパスワード。英大文字・小文字・数字を混ぜた12文字以上のものを設定し、どこかにメモしておきます。
   - 「Region」: 自分の場所に近いリージョンを選びます。日本からなら「Northeast Asia (Tokyo)」または「Southeast Asia (Singapore)」を選ぶとアクセスが速くなります。
4. 「Create new project」ボタンをクリックします。
5. 「Setting up your project...」と表示され、準備が始まります。1〜2分かかります。画面がプロジェクトのダッシュボードに変わるまで待ちます。

### 2-3. データベースのテーブルを作成する

アプリが使うデータの「箱（テーブル）」を作ります。GitHub に置いてある設定ファイルを使います。

1. `https://github.com/あなたのユーザー名/bookshelf-app` をブラウザで開きます。
2. ファイルの一覧の中から「supabase」というフォルダをクリックします。
3. 「schema.sql」というファイルをクリックします。
4. ファイルの中身（SQLコード）が表示されます。画面の右上あたりにある「Copy raw file」（または「Raw」ボタンを押してから Ctrl+A → Ctrl+C でコピー）ボタンでコード全体をコピーします。
   - 「Copy raw file」が見当たらない場合: 「Raw」ボタンをクリックすると、コードだけのページが開きます。そのページ上で Ctrl+A（Mac は Command+A）を押して全選択し、Ctrl+C（Mac は Command+C）でコピーします。
5. Supabase のダッシュボードに戻ります。
6. 左側のメニューから「SQL Editor」（データベースのアイコンのそば）をクリックします。
7. 「New query」ボタンをクリックします。
8. 画面中央の入力エリアにカーソルを置き、先ほどコピーしたコードを貼り付けます（Ctrl+V、または Mac は Command+V）。
9. 右上の「Run」ボタン（または Ctrl+Enter）をクリックします。
10. 画面下部に「Success. No rows returned」のようなメッセージが表示されれば成功です。

### 2-4. Supabase の URL と API キーを確認する

後の手順で必要になるので、2つの情報をメモしておきます。

1. 左側メニュー一番下の歯車アイコン「Settings（設定）」をクリックします。
2. 左側のサブメニューから「API」をクリックします。
3. 「Project URL」というセクションがあります。
   - 「URL」の欄に `https://xxxxxxxxxxxxxxxx.supabase.co` という形式のアドレスが表示されています。この値をコピーしてメモ帳などに貼り付けておきます（これが「Supabase URL」です）。
4. その下に「Project API keys」というセクションがあります。
   - 「anon」「public」と書かれた行があります。その右側に長い文字列（キー）があります。「Copy」ボタンをクリックしてコピーし、メモ帳などに貼り付けておきます（これが「Supabase Anon Key」です）。
   - 注意: 「service_role」と書かれた行のキーはコピーしません。「anon」の行だけです。

## ステップ 3: Vercel でアプリを公開する

Vercel はアプリをインターネット上に公開するサービスです。

### 3-1. Vercel アカウントを作成する

1. ブラウザで `https://vercel.com` を開きます。
2. 「Sign Up」ボタンをクリックします。
3. 「Continue with GitHub」をクリックします。
4. GitHub のログイン画面が表示された場合はログインします。
5. 「Authorize Vercel」と表示されたら「Authorize Vercel」ボタンをクリックします。
6. 「What's your name?」など、いくつか質問が出ます。名前を入力し、プランは「Hobby（無料）」を選択して進みます。
7. Vercel のダッシュボードが開けば登録完了です。

### 3-2. プロジェクトをインポートして環境変数を設定する

1. Vercel のダッシュボードで「Add New...」ボタン（または「+ New Project」）をクリックします。
2. 「Import Git Repository」という画面が表示されます。
3. GitHub と連携していない場合は「Connect GitHub」などのボタンが表示されます。クリックして GitHub との連携を承認します。
4. 自分のリポジトリ一覧の中から「bookshelf-app」を探し、右側の「Import」ボタンをクリックします。
   - 一覧に表示されない場合は「Adjust GitHub App Permissions →」というリンクをクリックし、GitHub 側でリポジトリへのアクセスを許可してから戻ってきます。
5. 「Configure Project」という設定画面が開きます。
6. 「Framework Preset」が「Next.js」になっていることを確認します（自動で選択されているはずです）。
7. 「Environment Variables」というセクションまで下にスクロールします。
8. 1つ目の環境変数を追加します。
   - 「Name」の欄に `NEXT_PUBLIC_SUPABASE_URL` と入力します（すべて大文字です）。
   - 「Value」の欄に、ステップ 2-4 でメモした「Supabase URL」（`https://xxxxxxxx.supabase.co` 形式）を貼り付けます。
   - 「Add」ボタンをクリックします。
9. 2つ目の環境変数を追加します。
   - 「Name」の欄に `NEXT_PUBLIC_SUPABASE_ANON_KEY` と入力します（すべて大文字です）。
   - 「Value」の欄に、ステップ 2-4 でメモした「Supabase Anon Key」（長い文字列）を貼り付けます。
   - 「Add」ボタンをクリックします。
10. 2つの環境変数が表示されていることを確認します。
11. 「Deploy」ボタンをクリックします。
12. ビルド（アプリの組み立て）が始まります。2〜5分かかります。画面に進行状況が表示されます。
13. 「Congratulations!」と大きく表示され、アプリの画面プレビューが見えれば成功です。
14. デプロイされたアプリの URL（例: `https://bookshelf-app-あなたのID.vercel.app`）が表示されます。この URL をメモしておきます。「Visit」ボタンをクリックすると実際のアプリが開きます。

---

## ステップ 4: Supabase の認証設定を完成させる

アプリが正しくログインできるよう、Supabase 側にアプリの URL を登録します。

1. Supabase のダッシュボードに戻ります（`https://supabase.com/dashboard` をブラウザで開きます）。
2. 先ほど作ったプロジェクト（`bookshelf-app`）をクリックして開きます。
3. 左側のメニューから「Authentication」（鍵のアイコン）をクリックします。
4. 左側のサブメニューから「URL Configuration」をクリックします。
5. 「Site URL」という入力欄があります。現在入力されている内容を削除し、ステップ 3-2 でメモした Vercel の URL（例: `https://bookshelf-app-あなたのID.vercel.app`）を入力します。末尾にスラッシュ（/）は不要です。
6. 「Save」ボタンをクリックします。
7. 少し下にスクロールすると「Redirect URLs」というセクションがあります。
8. 「+ Add URL」ボタンをクリックします。
9. 同じ Vercel の URL（例: `https://bookshelf-app-あなたのID.vercel.app`）を入力します。
10. 「Save」ボタンをクリックします。

---

## ステップ 5: 動作確認をする

1. ステップ 3-2 でメモした Vercel の URL をブラウザで開きます。
2. アプリのログイン画面が表示されます。
3. 「サインアップ」または「アカウントを作成」などのリンクをクリックします。
4. メールアドレスとパスワードを入力してアカウントを作成します。
5. メールアドレスに確認メールが届いた場合は、メール内のリンクをクリックして認証を完了します。
6. ログインしてアプリのメイン画面が表示されれば、セットアップは完了です。

# パート 2: 日常の使い方

---

## 基本的な使い方

- ブラウザで Vercel の URL（例: `https://bookshelf-app-あなたのID.vercel.app`）を開きます。
- メールアドレスとパスワードでログインします。
- スマートフォンのブラウザからでも同じ URL でアクセスできます。
- ログインは通常、一定期間（1週間程度）は維持されます。

## アプリの主な機能

- **本の登録**: タイトル、著者名、ISBN、ステータス（読みたい / 読書中 / 読んだ）、評価、感想を記録できます。
- **本棚の場所管理**: 「リビングの棚」「書斎」などの場所を登録し、本に紐付けることができます。
- **プロジェクトと参照メモ**: 読書メモをプロジェクトごとに管理できます。

# パート 3: トラブルシューティング

---

## よくある問題と対処法

### アプリが開かない / 「This page could not be found」と表示される

- Vercel の URL が正しいか確認します。
- Vercel のダッシュボード（`https://vercel.com/dashboard`）を開き、プロジェクトが「Ready」状態になっているか確認します。
  - 「Error」と表示されている場合は、プロジェクトをクリックし「Deployments」タブを開いて、赤くなっているデプロイをクリックすると原因が表示されます。

### ログインできない / 「Invalid login credentials」と表示される

1. メールアドレスとパスワードを間違えていないか確認します。
2. 「パスワードを忘れた場合」のリンクからパスワードリセットを試みます。
   - メールアドレスを入力すると、リセット用のリンクがメールで届きます。
   - メールが届かない場合は迷惑メールフォルダを確認します。

### サインアップ後にメールが届かない

1. 迷惑メールフォルダを確認します。
2. 「no-reply@mail.supabase.io」からのメールを探します。
3. メールが届かない場合は Supabase のダッシュボードで確認できます。
   - 「Authentication」>「Users」を開くと、登録されたユーザーの一覧が表示されます。
   - ユーザーが表示されているが「Confirmed」になっていない場合は、右側の「...」ボタンから「Confirm email」を選択してメール認証をスキップできます。

### ログイン後にエラーが出る / 「Invalid Redirect URL」と表示される

Supabase の認証設定が正しくない可能性があります。

1. Supabase のダッシュボードを開きます。
2. 「Authentication」>「URL Configuration」を開きます。
3. 「Site URL」と「Redirect URLs」に正しい Vercel の URL が設定されているか確認します。
4. URL の末尾にスラッシュ（/）が入っていないか確認します（入っている場合は削除して保存します）。

### 本のデータが保存されない / 表示されない

1. ログインが維持されているか確認します（画面右上にユーザー名やメールが表示されているか）。
2. ブラウザを更新（F5 または Ctrl+R）して再試行します。
3. 問題が続く場合は Supabase のデータを確認します。
   - Supabase ダッシュボード >「Table Editor」をクリックし、テーブルが存在するか確認します。
   - テーブルが表示されない場合は、ステップ 2-3 のSQL実行が失敗している可能性があります。改めて SQL Editor で schema.sql の内容を実行してみます。

### アプリの動作が確認できているか簡単にチェックする方法

1. ブラウザで Vercel の URL を開きます。ログイン画面が表示されれば Vercel のデプロイは正常です。
2. ログインできれば Supabase の認証は正常です。
3. 本を1冊追加してページを更新しても表示されれば、データベース（Supabase）への接続も正常です。

# パート 4: アプリを最新版に更新する

---

## 自動更新の仕組み

このアプリは、GitHub のコードが更新されると Vercel が自動的に新しいバージョンをビルドして公開する設定になっています。

- 自分が GitHub でコードを変更した場合 → 数分後に Vercel が自動的に最新版を公開します。
- 自分では何もしなくても更新が反映されます。

## 元のリポジトリで更新があった場合に自分のフォークに取り込む

元のリポジトリ（`https://github.com/jasmonicj/bookshelf-app`）でアプリが更新された場合、その変更を自分のフォークに取り込む手順です。

1. ブラウザで `https://github.com/あなたのユーザー名/bookshelf-app` を開きます。
2. ファイル一覧の上あたりに「This branch is X commits behind jasmonicj/bookshelf-app」というメッセージが表示されることがあります。
3. そのメッセージの横にある「Sync fork」ボタンをクリックします。
4. 「Update branch」ボタンをクリックします。
5. 「Successfully fetched and fast-forwarded from upstream」などのメッセージが表示されれば完了です。
6. 自動的に Vercel がビルドを開始し、数分後に最新版が公開されます。

## Vercel でのデプロイ状況を確認する方法

1. `https://vercel.com/dashboard` を開きます。
2. 「bookshelf-app」プロジェクトをクリックします。
3. 「Deployments」タブをクリックします。
4. 一番上のデプロイの状態が「Ready」（緑）であれば正常に公開されています。
5. 「Building」（黄）と表示されている場合はビルド中です。数分待ちます。
6. 「Error」（赤）と表示されている場合は、そのデプロイをクリックして詳細を確認します。
